# ai-reflect

让 AI 编程助手记住你是谁。

每开一个新会话，AI 对你的了解都归零。你的技术水平、说话方式、踩过的坑、上周才纠正过它的偏好，全部要从头再来一遍。ai-reflect 把"对你的了解"沉淀成一份跨工具共享的画像，每天在后台增量更新，并按各个工具自己的规矩写回它们的配置，让 Claude Code、Codex、Hermes 这些工具一开口就更贴近你。

它不是聊天记录备份，也不是又一个向量数据库。它做的是一件更小但更难的事：从你的真实对话里提炼出少量、准确、会过期的判断，并且不断自我修正和瘦身，不让这份画像越长越臃肿。

## 灵感来源

这个项目的想法不是凭空来的。几条公开的研究和实践把同一个方向指了出来：

- **斯坦福的 Generative Agents** 提出了 memory stream 加 reflection 的机制：智能体定期回看零散记忆，提炼成更高层的判断，再把判断本身也纳入下一轮回看。ai-reflect 的"三步反思"直接借了这个思路，并加了一条强制证伪的问题来对抗自我确认偏见。
- **Letta（前身 MemGPT）的 sleep-time compute** 论证了一件事：很多上下文是提前就知道的，与其等用户提问时再算，不如在空闲时先把它想透。ai-reflect 的每日后台反思就是这个理念在单机个人场景下的落地。
- **Claude Code 自己的 dream 机制**（社区从发行包里提取出的一段 agent 提示词）走的也是这条路：空闲时对记忆文件做一次反思式回顾，合并新信号、删掉被推翻的旧事实、保持索引精简。它和本项目的设计不谋而合，也印证了"合并而非堆叠""相对日期转绝对""索引不是垃圾堆"这些原则是对的。

ai-reflect 没有复制上述任何项目的代码或提示词，只是把它们共同指向的方法，做成一个能在你自己机器上、跨多个工具、可回滚、会自我审计的闭环。

## 它解决什么

- **每日反思**：增量读各工具的新对话，提炼你的语言风格、习惯、技术水平和协同节奏，合并进一份画像。
- **项目复盘**：持续记录你和 AI 一起解决了什么难题、走了哪些弯路、最后怎么解决的，沉淀成可复用的经验。
- **配置自优化**：按各工具的约定（Claude 用 CLAUDE.md、Codex 用 AGENTS.md、Hermes 用 SOUL.md）补充行为提示和技能路由，让长期闲置的能力被合理调起来。
- **进化与瘦身**：你的水平在涨，旧的引导就该收掉。它会删过时、合冗余、控 token，让画像始终短而准。
- **按需报告**：你说一声就出一份基于真实数据的报告，覆盖使用频率、功能、时长、token 成本、做过的项目、遇到的问题及解法、你和 AI 各自的成长。
- **沟通风格由你定**：AI 用什么语气跟你说话，是你设的，随时能改。它不会偷偷模仿你的措辞，原因见下面的"注意事项"。

## 为什么是这个设计

几个刻意的取舍，每一个都对应一个真实的失败模式：

**判断交给模型，确定的事交给代码。** 提炼洞见、判断偏好稳不稳，这些需要判断力，写进 Skill 交给 LLM，模型升级它就跟着变强。读数据库、扫密钥、加窗口、git 操作，这些必须可靠、不能靠猜，写成 Python。

**闭环要能自己验证对错。** 系统会统计"你纠正 AI 的频率"随时间怎么变。但纠正变少不一定是 AI 变好，也可能是你放弃了、或者干脆不用了。所以它会同时看你的参与度，纠正和参与一起降时不当成功，而是主动问你一句，绝不闷头给自己发奖励。

**默认不直接改，先给你看。** 自动改配置默认走草稿模式，改动先攒进待审目录，你点头才合并。信得过了再切直接写入，每次都带备份和变更日志，随时能回滚。

**省 token 的工具自己不能烧 token。** 增量读取有水位线，每轮还有消息数和天数的硬上限，多日不开机也不会某一轮把上下文撑爆。没有新对话的日子，它只记一次心跳就收工。

## 安装

前置条件：Python 3.9 或更高；可选 git（用于回滚和同步）；可选 `pip install mcp`（用于跨工具实时读画像）。

```bash
git clone https://github.com/<your-name>/ai-reflect.git
cd ai-reflect
python engine/install.py
```

安装器会带你走完一轮交互配置：

1. 扫描本机已装的 AI 工具，逐个问你要不要接入（默认不接，你明确同意才接）。
2. 选同步方式：git 私有远程、云盘文件夹、或手动导出包。
3. 选回滚方式：git 或本地备份。
4. 定每日反思的运行时间。
5. 指定 AI 的初始沟通风格，可以留空。
6. 录入额外敏感词（客户名、项目代号这类没有固定特征、正则抓不住的），写画像和报告前会被替换掉。

装完后它会注册一个操作系统级的定时任务作为主心跳，不依赖任何图形界面常驻。

### 作为 Claude Code 插件

把仓库放进 Claude Code 的插件目录，即可使用四个命令：`/reflect` 立刻跑一轮、`/reflect-setup` 重新扫描配置、`/reflect-report` 出报告、`/reflect-style` 改沟通风格。同时会启用 ai-reflect-memory 这个 MCP server，供其他支持 MCP 的工具实时只读画像。

## 日常使用

平时不用管它，定时任务会自己跑。草稿模式下，改动进 `~/.ai-reflect/local/review/`，你看过再合并。需要时手动调：

```bash
# 立刻跑一轮反思
/reflect

# 出一份周报（也可以 月报 / 2026 / 自定义范围）
/reflect-report 周报

# 改沟通风格
/reflect-style 简洁直接，少用术语
```

卸载：`python engine/uninstall.py`，会移除定时任务，并问你是否保留画像。

## 工作原理

```
判断层（LLM）       skills/daily-reflect/   提炼洞见、画像演化、复盘、风格建议
确定性层（Python）  engine/                 读取、脱敏、反馈信号、可回滚写入、心跳、MCP
心跳（操作系统级）   Task Scheduler / cron   python -m engine daily
```

数据分两层存放，这是跨设备能用的关键：

```
~/.ai-reflect/
  synced/   跟你走、可同步：画像、分面画像、复盘、偏好、变更日志
  local/    绑本机、绝不同步：本机路径、水位线、device_id、备份、报告
```

水位线、本机绝对路径、设备标识全部留在 local，所以 A 机的进度和路径不会污染 B 机。换设备时拉下 synced，在新机重新扫描重建 local 即可。

更细的设计、安全模型、同步方式分别见 [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)、[docs/SECURITY.md](docs/SECURITY.md)、[docs/SYNC.md](docs/SYNC.md)。这套方案在打包前经过两轮对抗性审计（一轮查逻辑闭环、一轮查安全漏洞），过程和修复记录在 [docs/AUDIT.md](docs/AUDIT.md)。

## 注意事项

- **只覆盖本地工具的对话。** 网页版 claude.ai 的数据在服务端，本地拿不到，所以抓不到。
- **沟通风格是你指定的，AI 不自动模仿你。** 这是有意为之。如果让 AI 自动学你的措辞再用回给你，你会被它带着更频繁用那些词，下一轮它又看到"你更常用了"，于是越学越像，最后你俩的语言风格趋同，而它已经分不清哪些是你本来的、哪些是它自己的回声。把风格交给你手动控制，这个雪球就从根上不存在。
- **正则脱敏不是万能的。** 有固定特征的密钥（API key、私钥、token）能可靠抓住，但没特征的敏感词（某个客户名）正则挡不住，要靠你在安装时录入的敏感词表，外加一条硬规则：画像只写抽象结论，绝不逐字摘录对话原文。
- **git 历史是永久的。** 万一有敏感信息漏网并推到了远程，需要用 `git filter-repo` 重写历史，单纯删文件没用。
- **云盘同步等于把画像交给云厂商。** 在意这点就选 git 私有远程或手动导出包。
- **它会改你其他工具的配置文件。** 所有自动写入的内容都包在 `<!-- ai-reflect:auto BEGIN/END -->` 注释之间，一眼能认出来，也能手动删掉。第三方 skill 的正文它永远不碰，最多在备份后改一下触发描述。
